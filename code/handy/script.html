# script标签中的async和defer属性

script

浏览器在解析HTML代码的时候，如果遇到一个没有任何属性的script标签，就会暂停解析HTML，先发送网络请求获取该js脚本的代码并解析，然后让js引擎执行该代码，当js代码执行完毕后恢复解析。整个过程如下图所示

浏览器解析到js脚本时会阻塞解析html模板,问题: 假如js脚本加载或执行时间过长都会导致白屏

async script

async表示异步，但浏览器遇到带有async属性的script时，请求该脚本的网络请求是异步的，不会阻塞浏览器解析HTML，一旦网络请求回来之后，分一下两种情况： -此时HTML还没有解析完，浏览器会暂停解析HTML，先让js引擎执行代码，执行完毕后再解析HTML； -此时HTML代码解析完成，直接执行js代码；

​ 获取js文本时不会停止浏览器解析html,但是执行js脚本会停止浏览器对js的解析. 由于浏览器解析html文本时间不确定,获取/js的事件也是不确定.因此可能会对html照成堵塞,也会可能有执行js脚本获取不到dom的情况

defer script

defer表示延迟，当浏览器遇到带有defer属性的script的时候，获取该脚本的网络请求也是异步的，不会阻塞浏览器解析HTML，一旦网络请求回来之后，如果此时HTML代码还没有解析完，浏览器不会暂停解析HTML代码并执行js代码，而是等待HTML解析完毕以后再执行js代码，如图：

defer是不会照成堵塞html解析的,执行js脚本的时候放在了解析完html的后面.如果存在多个defer script标签，浏览器会保证他们按照在HTML中出现的顺序执行，不会破坏js脚本之间的依赖关系。

const jwt = require('jsonwebtoken')
const app = express()

app.use(express.json())

app.post('/login', (req: Request, res: Response) => {
  const data = req.body
  if (data.username && data.password) {
    let selSql = `select * from users where username='${data.username}' and password=${data.password}`
    db.query(selSql, (err: any, results: any) => {
      if (err) return res.send({ message: err })
      if (!results.length) return res.status(500).send({ message: '用户不存在' })
      if (!results[0].state) return res.send({ message: '用户未开启状态' })
      let token = jwt.sign({username:data.username,password: data.password},'lililixin')
      res.send({
        state: '登录成功',
        token
      })
    })
  }
})

# React脚手架使用eventbus

当页面足够复杂时会出现很多个组件，这是当一个组件数据和嵌套较深的组件数据会有互相牵扯时这时候再使用传统的层层传递显然是足够麻烦，这时就可以使用工具bus总线

npm下载 hy-event-store

npm i hy-event-store

这里我们的实例分为三个组件 父组件App.jsx 子组件 Home.jsx , 孙子组件HomeBanner.jsx

在父组件中引入Home组件

import React, { Component } from 'react'
import Home from './Home'
class App extends Component {
  render() {
    return (
      <div>App
        <Home/>
      </div>
    )
  }
}
export default App

在Home组件中引入HomeBanner

import React, { Component } from 'react'
import HomeBanner from './HomeBanner'
class Home extends Component {
  render() {
    return (
      <div>Home
        <HomeBanner/>
      </div>
    )
  }
}
export default Home

场景：在HomeBanner中触发方法修改App组件中数据

首先需要创建js文件这里叫event-bus.js

import { HYEventBus } from 'hy-event-store' //引入
const eventBus = new HYEventBus() //创建实例
export default eventBus //导出

在HomeBanner引入eventBus

import React, { Component } from 'react'
import eventBus from './event-bus' //引入
class HomeBanner extends Component {
  prevClick() {
     //触发eventBus.emit方法（参数解析： 父级组件需要触发方法的名字，接下来依次是参数）
    eventBus.emit("bannerPrev", "kobe",22)
  }
  render() {
    return (
      <div>
        HomeBanner
        <button onClick={e => this.prevClick()}>传导bus</button>
      </div>
    )
  }
}

export default HomeBanner

在父级组件中 引入eventBus 进行 监听/卸载


import React, { Component } from 'react'
import Home from './Home'
import eventBus from './event-bus' //引入

class App extends Component {
  constructor() {
    super()
    this.state = {
      name: '',
      age: undefined
    }
  }
  // 监听 （在DOM挂载后 进行监听） 参数： 需要监听的函数名，执行函数，this指向 （可选 需要指向当前this的话执行函数可写成箭头函数或者bind修改）
  componentDidMount() {
    eventBus.on('bannerPrev',this.bannerPrev,this)
  }
  bannerPrev(name,age){ 
    console.log(name,age,'监听成功')
    this.setState({name,age},()=>{console.log(this.state)})
  }
  // 移除 （这里使用的是根组件，当然不会卸载，但是假如使用的是其他组件的话要尽可能去考虑这方面）
  componentWillUnmount() {
    eventBus.off('bannerPrev',this.bannerPrev)
  }
  render() {
    return (
      <div>App

        <Home/>
      </div>
    )
  }
}

export default App
